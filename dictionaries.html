1. Introduction

Dictionaries allow CML to be understood by machines. Much of physical science is managed through the dictionary mechanism. We find terms and units relating to a aspect of science (such as heat of formation, melting point, point group) and create entries for these items in a dictionary.

The entries can consist of just a unique id (within the dictionary's namespace) and some human-understandable definition but we highly encourage more information to be given. For instance what are the units, are there upper and lower bounds, what is the type of the data (string, integer, float etc).

Different programs sometimes produce data with the same label but a different interpretation; does density mean electron density or physical density? Therefore each computational chemistry code will have its own dictionary and then the community can then decide to group particular concepts together.

Where concepts are defined by the CML schema they SHOULD NOT be redefined using the dictionary mechanism.

Where concepts are defined by standard dictionaries these entries SHOULD be referenced, rather than redefining the concept in another dictionary.

2. Dictionaries

2.x Namespace

Dictionaries MUST have a namespace attribute, which MUST be a valid URI. The namespace defines the scope within which the entry terms are unique. A dictionary's namespace URI SHOULD resolve to a representation of the dictionary. A dictionary's namespace URI SHOULD end with either a '/' character or a '#' character so that terms may be referenced by appending them to the URI.

In order for namespaces to be resolvable a CML document SHOULD NOT contain more than one dictionary.

2.x Convention

Dictionaries SHOULD specify a convention. The RECOMMENDED convention is cml-conv:dictionary.

2.x Title

Dictionaries SHOULD have a title attribute intended for human-readability.

2.x Description

Dictionaries SHOULD have a single description child, the contents of which describe the domain of the dictionary.

2.x Prefix

Dictionaries SHOULD have a dictionaryPrefix attribute specifying the recommended prefix to use when referencing dictionary entries. The dictionaryPrefix MUST be a valid [XML QName prefix -- i.e. cannot be 'xml'], and SHOULD be unique within the CML domain.

2.x. Conformance

Dictionaries conforming to the cml-conv:dictionary convention MUST contain at least one entry.

Dictionaries conforming to the cml-conv:dictionary convention MUST have a title.

2.x Example

<?xml version="1.x" encoding="UTF-8"?>
<dictionary xmlns="http://www.xml-cml.org/schema"
    xmlns:cml-conv="http://www.xml-cml.org/convention/"
    namespace="http://www.xml-cml.org/dictionary#"
    convention="cml-conv:dictionary"
    title="My dictionary"
    dictionaryPrefix="my">

    <description>

    </description>

    <!-- entry goes here -->

</dictionary>


<?xml version="1.x" encoding="UTF-8"?>
<dictionary xmlns="http://www.xml-cml.org/schema">

</dictionary>

3. Dictionary Entries

3.x Id

A dictionary entry MUST have an id attribute, the value of which MUST be unique within the scope of the dictionary.

TODO: The id MUST start with a letter, and MUST only contain letters, numbers, dot, hyphen or underscore.

The id is used to reference specific entries in a dictionary.

3.x Term

A dictionary entry MUST have a term attribute, the value of which provides a unique nounal phrase linguistically identifying the object.

The value of the term attribute SHOULD only contain characters from the ASCII subset (codepoints 32-127). Non-ASCII characters (e.g. greek letters, degree symbol) SHOULD be spelt out using the ASCII subset (e.g. alpha, degree).

3.x Definition

A dictionary entry MUST have a single definition child, the content of which provides a nounal phrase defining of the subject of the entry more verbosely than the term.

Children of the definition element MUST be in the XHTML namespace.

3.x Description

An entry MAY have a single description child, the content of which provides further information regarding the term, including, but not limited to: examples, human-readable semantics and hyperlinks to other useful resources.

Children of the description element MUST be in the XHTML namespace.

3.x Unit type

When applicable to the concept defined, an entry SHOULD have a unitType attribute referencing a unit type (e.g. temperature) in a dictionary of unit types.

3.x Default unit

When applicable to the concept defined, an entry SHOULD have a unit attribute defining the default unit used to qualify values referenced against this entry, referencing a unit (e.g. Kelvin) in a dictionary of units.

3.x Example

<?xml version="1.x" encoding="UTF-8"?>
<dictionary xmlns="http://www.xml-cml.org/schema" xmlns:x="http://www.w3.org/1999/xhtml">
    <entry id="" term="">
        <definition>

        </definition>

        <description>
            <x:p>

            <x:/p>
        </description>
    </entry>
</dictionary>

4. Standard Dictionaries

Standard (widely used) dictionaries SHOULD be namespaced under the http://www.xml-cml.org/dictionary/ domain, and registered with the xml-cml.org site.
